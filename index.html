Linux——用户管理

2019-03-16 Comments Linux

用户/组基本概念

users and groups:

  • 系统上每个进程（运行的程序）都是特定用户运行的
  • 每个文件都是由一个特定的用户拥有
  • 访问文件和目录是由用户限制的，特定文件是只能特定用户访问（用户访问文件其实是某一个进程在访问，并不是用户）
  • 与运行的进程相关联的用户确定该进程可访问的文件和目录

    查看当前登陆的用户信息

    [root@localhost ~]# id

查看文件的owner

[root@localhost ~]# ls -l /var/www/html

查看运行进程的username

[root@localhost ~]# ps aux |less

用户管理

groupadd,groupdel,useradd,usermod,userdel,passwd,chage

[root@localhost ~]# groupadd wang //创建wang组
[root@localhost ~]# tail /etc/group //查看组，可以看见刚刚新建的wang组

[root@localhost ~]# groupadd wang -g 520
//可以自定义gid 为520
//yang:x:520:

[root@localhost ~]# groupdel wang //删除组

用户

[root@localhost ~]# useradd yogurt
[root@localhost ~]# grep ‘yogurt’ /etc/passwd /etc/shadow /etc/group //过滤yogurt用户的信息 

/etc/passwd:yogurt:x:500:500::/home/yogurt:/bin/bash
/etc/shadow:yogurt:$6$G.Bp3shG$2kpG4fswPs4QsSAZicTNe/g5fzUxSfkMIju3X8Mo90uDIfc/VLS8ofCt2M6EEDyhU6bp.19t1hAP9If.LX/Us1:17966:0:99999:7:::
/etc/group:yogurt:x:500:

小结：如果创建一个用户时，未指定任何组（主组或附加组），系统会创建一个和用户名相同的组作为用户的Primary Group-主组，而附加组可以有多个，创建用户时指定-G x，y，z即可。如果用户已存在，可以加入附加组，usermod、gpasswd

创建用户 指定选项

[root@localhost ~]# useradd wang -u 123 //创建用户wang，指定uid
[root@localhost ~]# useradd wang -d /aaa //指定家目录
[root@localhost ~]# useradd wang -s /sbin/nologin //创建用户并指定shell，例如我指定了bash的话，那这个用户就只能从bash登陆
[root@localhost ~]# useradd wang -G hr,it,haha //创建用户，指定附加组

删除用户

[root@localhost ~]# userdel wang //删除用户wang，但不删除用户家目录和mail spool
[root@localhost ~]# ls -l -d /home/yogurt
[root@localhost ~]# ls -l /var/spool/mail/yogurt

[root@localhost ~]# userdel -r user1 //删除用户user1，同时删除用户家目录和mail spool

用户密码

管理员root可以给任何用户设置密码
[root@localhost ~]# passwd user2

而用户只能给自己修改密码
[root@localhost ~]# passwd

组成员管理

只针对已存在的用户

[root@localhost ~]# usermod -G hr wang //覆盖原有的附加组

[root@localhost ~]# usermod -G ab,cd wang

[root@localhost ~]# usermod -aG hr wang //给hr用户增加新的附加组wang

但是一般不用这个处理组，用下边这个gpasswd

[root@localhost ~]# gpasswd -a yogurt lueluelue//把用户添加到组
[root@localhost ~]# gpasswd -M yogurt,da lueluelue //将yogurt和da都加入lueluelue组
[root@localhost ~]# gpasswd -d yogurt lueluelue //从组中删除用户

其他选项管理

[root@localhost ~]# usermod -s /sbin/nologin yogurt

no shell

/sbin/nologin 用户无法登陆系统，实现管理，仅作为运行进程的用户，访问FTP的用户。 //安全的用户，如果不打算给用户太多权限，就设置为这个shell，以保安全

/bin/bash
登陆系统，实现管理

shell是用户登录后运行的第一个程序 ，你给它什么，就执行什么

举个栗子：
vim /etc/passwd 把yogurt的shell改为/sbin/poweroff(关机命令，通过which poweroff找到命令的位置)，一登陆yogurt时，就会执行关机命令。

useradd参照文件
有两个文件会影响useradd创建用户。

[root@localhost ~]# vim /etc/login.defs
[root@localhost ~]# vim /etc/default/useradd
通常改SHELL=/sbin/nologin

sudo

普通用户提权

法一

  • [wangwang@localhost ~]# useradd ab 可能会说你没权限
  • 于是[root@localhost ~]# su - root
    输入密码即可变成root

    需要注意的是 这个是建立在你知道root密码的前提下

法二

以root身份授权普通用户
[root@localhost ~]# vim /etc/sudoers

可以看到wheel组可以使用sodu命令

于是把用户添加到wheel组：
[root@localhost ~]# useradd yangyang -G wheel
可以查看一下:
[root@localhost ~]# id yangyang

此时yangyang用户就具有了变成管理员的能力，但是每次执行命令都需要sudo。

[yangyang@localhost ~]# sudo useradd dawang

Linux——Shell

2019-03-13 Comments Linux

初识shell

shell是系统的用户界面，提供了用户与内核进行交互操作的一种接口。它接收用户输入的命令并把它送入内核去执行。实际上shell是一个命令解释器，它解释由用户输入的命令并且把它们送到内核。不仅如此，shell有自己的编程语言用于对命令的编辑，它允许用户编写由shell命令组成的程序。shell编程语言具有普遍编程语言的很多特点，比如它也有循环结构和分支控制结构等，用这种编程语言编写的shell程序与其他应用程序具有同样的效果。

逻辑漏洞定义

逻辑漏洞是指由于程序逻辑不严谨或逻辑太复杂，导致一些逻辑分支不能正常处理或处理错误。

通俗讲：一个系统的功能太多后，程序开发人员就难以思考全面，对其他地方可能有遗漏，或者未能正常处理，从而导致逻辑漏洞。逻辑漏洞也可以说是程序开发人员的思路错误、程序开发人员的逻辑存在漏洞。

同源策略

同源策略是一种约定，是浏览器最核心最基本的安全功能。如果缺少同源策略，浏览器的正常功能可能会被影响。可以说web是建立在同源策略的基础之上的。

对于客户端Web安全的学习与研究，深入理解同源策略很重要。浏览器只是针对同源策略的一种实现。

浏览器的同源策略限制了来自不同源的document或脚本，对当前的document读取或设置某些属性。

关于抓肉鸡

2019-02-16 Comments 随笔

‘抓鸡’抓的是可以背黑客控制的计算机，”肉鸡”可以是各种系统,如2003,2000,2008,xp,win7,win8,linux,unix等；更可以是一家公司\企业\学校甚至是政府军队的服务器。一旦成为了肉鸡之后黑客们就可以像操作自己的电脑那样来操作它而不被对方所发觉。
黑客们经常用80,8080,135,445,1433,1521,3306,3389 ,4899,5900之类的端口来抓鸡。现在许多人把有WEBSHELL 权限的机器也叫肉鸡。
黑客们利用工具就能抓到肉鸡。
https://www.cnblogs.com/heikeboke/p/8043415.html
这篇文章解释了3389“肉鸡”和“4489肉鸡”。

不过抓鸡是犯法的奥，如果是内部测试玩玩的话，还是可以的。

抓鸡，常见的就是利用系统弱口令进行暴力破解，从而拿到系统权限的一个过程。新手就可以玩，不需要高深技术水平。

通过扫描多个网段开放的下面这些端口，使用暴力破解软件进行破解:

  1. 远程桌面服务，默认3389端口
  2. MYSQL数据库，默认3306端口
  3. MSSQL数据库，默认1433端口
  4. FTP服务，默认端口21
  5. SSH服务，默认端口22
    and so on………………

首先呢要扫描ip，例如你想扫开启3306端口的主机，然后会扫到一批开启了3306端口的主机IP地址。扫ip的工具很多。网上都可以下载到。

然后将ip导入爆破工具里，导入用户字典和密码字典，从而成功登陆远程主机。DUbrute工具也是其一，这种爆破工具很多，功能也很多，掌握几个玩玩就可以了。

获得权限以后可以上传自己制作的木马或者其他操作。

CSRF漏洞

2019-02-07 Comments web

何为CSRF

CSRF（Cross-site request forgery）跨站请求伪造,通常缩写为CSRF或者XSRF，是一种对网站的恶意利用。与XSS非常不同，XSS利用站点内的信任用户，而CSRF则通过伪装成受信任用户的请求来利用受信任的网站。与XSS攻击相比，CSRF攻击往往不大流行（因此对其进行防范的资源也相当稀少）和难以防范，所以被认为比XSS更具危险性。——来自百度
跨站请求伪造就是在另一个网站向另一个网站(本该真正用户请求的网站)发送一个伪造的请求，而接受请求的网站以为该伪造的请求是用户发送来的，故将此请求执行，造成了CSRF攻击。

CSRF可以做什么

你这可以这么理解CSRF攻击：攻击者盗用了你的身份，以你的名义发送恶意请求。CSRF能够做的事情包括：以你名义发送邮件，发消息，盗取你的账号，甚至于购买商品，虚拟货币转账……造成的问题包括：个人隐私泄露以及财产安全。
