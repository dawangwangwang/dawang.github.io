CSRF漏洞

挖洞经验

主要用于越权操作，漏洞容易在有权限控制的地方。例如管理后台，会员中心，论坛帖子，交易管理，其中管理后台是高危的地方。一般CSRF很少被关注，所以还是蛮多程序存在这个漏洞的。

挖掘的时候：对几个非静态操作的页面抓包，看看是否有token，若没有token，再请求这个页面，不带referer。如果返回的数据还是一样，说明可能有CSRF漏洞。（黑盒挖掘）

还可以读代码，看核心文件有木有验证token和referer相关的代码。（白盒）

防范

管理员权限提交到服务器的数据也不一定就是完全可靠的，所以对csrf的防御有两点1）增加token/referer验证2）增加验证码

文件包含漏洞

php的文件包含就是可以直接执行包含文件的代码，分为本地文件包含，远程文件包含。文件包含漏洞大多可以直接获取webshell

文件包含函数有
include(),include_once(),//在包含文件时遇到错误下面的代码会继续执行
require(),require_once(),//在包含文件时遇到错误会直接报错退出

挖洞经验

文件包含漏洞大多出现在模块加载，模板加载以及cache调用的地方。比如传入模块名，然后拼接到文件路径中，而这个模块名是我们可控的，所以如果没有其他什么过滤的情况下 就可以包含文件了。

远程文件包含需要设置allow_url_include=On;
若测试代码为

Code

利用则在get请求url参数里面传入”http://ip/2.txt",2.txt是<?php phpinfo();?>。也可以用php输入输出流的方式利用，url输入1.php?a=php://input,post内容为<?php phpino();?>

文件包含截断

大多数的文件包含漏洞都是需要截断的。

%00截断

方法最老，受限于GPC和addslashes等函数的过滤，在开启GPC的情况下不可用。在PHP5.3之后的版本不能用了。

多个英文句号(.)和反斜杠(/)来截断，不受GPC限制，但是同样在PHP5.3版本后被修复。

?伪截断

?伪截断不受GPC和PHP版本影响，只要能返回代码给包含函数，就能执行。
在HTTP协议里，访问http://ip/1.txt 和访问http://ip/1.txt?.php返回结果一样。这时候webserver把问号之后的内容当成了请求参数。txt不在webserver里解析，参数对访问1.txt返回的内容不影响，实现了伪截断。

文件读取（下载）漏洞

文件读取漏洞和下载漏洞差不多。

挖掘经验

文件读取漏洞寻找：
一种：黑盒看看功能点对应的文件，再去读文件。
二种：搜索文件读取函数，file_get_content(),highlight_file(),fopen(),readfile(),fread(),fgetss(),fgets(),parse_ini_file(),show_source(),file(),还有一些其他功能函数也可以读取文件，例如文件包含函数include……，可以利用PHP输入输出流php://filter/读取文件

文件上传漏洞

文件上传漏洞是出现最早的漏洞，也最容易理解。代码都是写在文件里面然后执行的。若能把文件上传到管理员或网站不想让你上传的目录，那么就视为存在文件上传漏洞。

早期文件上传漏洞大多是没有限制文件格式导致可以上传文件，后来较多是黑名单过滤可以被绕过导致文件上传漏洞。

挖掘经验

文件上传漏洞较容易理解，一般应用能上传的点比较少，目前大部分web应用都是基于框架来写，上传的点自然也都是调用的同一个上传类，而上传函数只有move_uploaded_file()。所以代码审计时，最快就是搜索这个函数，然后看调用这个函数的代码有木有限制文件上传格式或者可以绕过，黑名单限制文件格式以及未更改文件名的方式，问题会多一些。在apache利用其向前寻找解析格式和IIS6的分号解析bug都可以

防范

  • 白名单方式过滤扩展名，用in_array或者===来对比扩展名
  • 文件上传后重命名，命名规则尽量不可预测。例如（时间戳+随机数）的md5值

代码执行漏洞

应用程序本身过滤不严，用户通过请求将代码注入执行。
sql注入是把sql语句注入到sql服务，php代码执行是把代码注入到应用中从而到webserver中执行。
该漏洞主要是由eval(),assert(),preg_replace(),call_user_func(),call_user_func_array(),array_map()等函数的参数过滤不严格导致

命令执行漏洞

漏洞可以执行系统或者应用命令（cmd，bash）
漏洞主要基于一些参数过滤不严导致，可以执行的命令函数有system(),exec(),shell_exec(),passthru(),popen()……反引号 `
也可以执行命令（实际上也是调用的shell_exec()函数）

防范

一： 使用PHP自带的命令防注入函数，escapeshellcmd()和escapeshellarg()，escapeshellcmd()是过滤整条命令，参数是一整条命令。escapeshellarg()则是拼命的使参数是字符串，包在双引号里。
二：命令执行函数的参数做白名单限制

变量覆盖漏洞

自定义参数值替换程序原有的变量值，通常需要结合其他功能
大多是由于函数使用不正确造成的，经常引发变量覆盖漏洞的函数有extract()函数和parse_str()和import_request_variables()

还有$$变量覆盖

$a=b
$$a=2

防范

  • 使用原始的变量，不进行变量注册，或者直接在代码中定义变量，再赋值
  • 验证变量是否存在

逻辑处理漏洞

程序在业务逻辑上面的漏洞，代码层和业务层（支付，找回密码，程序安装）

挖掘

挖掘逻辑漏洞一定要熟悉业务流程，值得关注的有程序是否可重复安装，修改密码是否可以越权修改其他用户，找回密码是否可以暴力破解和修改其他用户密码，cookie是否可预测可绕过

防范

深入熟悉业务逻辑，熟悉函数功能和差异
