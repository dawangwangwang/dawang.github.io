    可以看到input 的变量经过了 php 的intval处理
    invalintval 是防止程序溢出的，我们输入的大于 21 位的数字经过 PHP 处理的值一定发生了变化。
    输入的4200000000000000000000经过 PHP 的 intval 函数处理的结果为:9223372036854775807，
    所以我们输入的这个值经过 php 防止溢出处理后的实际值为9223372036854775807

    1.$_REQUEST的变量覆盖

    一段一段看奥

    Code

    上述代码把$_GET,$_POST,$_COOKIE传进来的值进行了正则匹配，如果包含了A-Z和a-z的字母就返回waf

    这里的$_REQUEST有一个特性，就是当GET和POST都存在同一个变量名的时候，只获取POST中的值，所以可以通过这个特性来绕过正则的匹配

    示例图片：

    接着往下分析

    Code

    $_SERVER[‘QUERY_STRING’]这里的bypass，这个点应该是比较常见的了，$_SERVER[‘QUERY_STRING’]` 获取的值是未经url编码的，所以直接编码一下就好了。

    接下来的呢：

    Code

    if(!(substr($_GET[‘yulige’], 32) === md5($_GET[‘yulige’])))，传进去是个数组的时候，值就为空，自然就相等了
