HEXO报错:TypeError: Cannot read property 'replace' of undefined

2019-02-07 Comments 报错

错的出现

先说这错咋出现的，前些日子编辑的一篇博客，但是忘记更新博客了，以至于前几天给博客换主题时报这个错时，我很懵，明明什么都没干。

后来试了网上很多方法，有人说时url的问题，有人直接建议把md和配置文件提出来，然后重新下载hexo。我试了好多种方法……

最后快崩溃了的时候有一个大哥提醒我了是不是更新的文章编写不规范？这我才去找我有没有更新文章……原来是我在更新主题的时候把一个之前我没更新的文章顺道更新了。导致了报错。

解决方法

我把不规范的那一篇博客先拿出来，再更新就好了。然后自己把博客编写规范再重挂上就好了。

最后 提醒一句：博客编写一定要规范阿！

代码执行

2018-11-21 Comments web

背景

当应用在调用一些能将字符串转化成代码的函数时(如PHP中的eval、assert)，没有考虑到用户是否能控制这个字符串，所以可能会造成执行漏洞。狭义的代码注入通常指将可执行代码注入到当前页面中，如php的eval函数，可以将字符串代表的代码作为php代码执行，当前用户能够控制这段字符串时，将产生代码注入漏洞（也称命令执行）。广义上的代码注入，可以覆盖大半安全漏洞的分类。

漏洞危害

执行任意代码
写入webshell
甚至控制服务器
……

常见代码执行的方式

  • ${}执行代码
  • eval()
  • assert()
  • preg_replace()
  • create_function()
  • array_map()
  • call_user_func()/call_user_func_array()
  • array_filter()
  • usort(),uasort()
