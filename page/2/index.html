关于抓肉鸡

2019-02-16 Comments 随笔

‘抓鸡’抓的是可以背黑客控制的计算机，”肉鸡”可以是各种系统,如2003,2000,2008,xp,win7,win8,linux,unix等；更可以是一家公司\企业\学校甚至是政府军队的服务器。一旦成为了肉鸡之后黑客们就可以像操作自己的电脑那样来操作它而不被对方所发觉。
黑客们经常用80,8080,135,445,1433,1521,3306,3389 ,4899,5900之类的端口来抓鸡。现在许多人把有WEBSHELL 权限的机器也叫肉鸡。
黑客们利用工具就能抓到肉鸡。
https://www.cnblogs.com/heikeboke/p/8043415.html
这篇文章解释了3389“肉鸡”和“4489肉鸡”。

不过抓鸡是犯法的奥，如果是内部测试玩玩的话，还是可以的。

抓鸡，常见的就是利用系统弱口令进行暴力破解，从而拿到系统权限的一个过程。新手就可以玩，不需要高深技术水平。

通过扫描多个网段开放的下面这些端口，使用暴力破解软件进行破解:

  1. 远程桌面服务，默认3389端口
  2. MYSQL数据库，默认3306端口
  3. MSSQL数据库，默认1433端口
  4. FTP服务，默认端口21
  5. SSH服务，默认端口22
    and so on………………

首先呢要扫描ip，例如你想扫开启3306端口的主机，然后会扫到一批开启了3306端口的主机IP地址。扫ip的工具很多。网上都可以下载到。

然后将ip导入爆破工具里，导入用户字典和密码字典，从而成功登陆远程主机。DUbrute工具也是其一，这种爆破工具很多，功能也很多，掌握几个玩玩就可以了。

获得权限以后可以上传自己制作的木马或者其他操作。

CSRF漏洞

2019-02-07 Comments web

何为CSRF

CSRF（Cross-site request forgery）跨站请求伪造,通常缩写为CSRF或者XSRF，是一种对网站的恶意利用。与XSS非常不同，XSS利用站点内的信任用户，而CSRF则通过伪装成受信任用户的请求来利用受信任的网站。与XSS攻击相比，CSRF攻击往往不大流行（因此对其进行防范的资源也相当稀少）和难以防范，所以被认为比XSS更具危险性。——来自百度
跨站请求伪造就是在另一个网站向另一个网站(本该真正用户请求的网站)发送一个伪造的请求，而接受请求的网站以为该伪造的请求是用户发送来的，故将此请求执行，造成了CSRF攻击。

CSRF可以做什么

你这可以这么理解CSRF攻击：攻击者盗用了你的身份，以你的名义发送恶意请求。CSRF能够做的事情包括：以你名义发送邮件，发消息，盗取你的账号，甚至于购买商品，虚拟货币转账……造成的问题包括：个人隐私泄露以及财产安全。

HEXO报错:TypeError: Cannot read property 'replace' of undefined

2019-02-07 Comments 报错

错的出现

先说这错咋出现的，前些日子编辑的一篇博客，但是忘记更新博客了，以至于前几天给博客换主题时报这个错时，我很懵，明明什么都没干。

后来试了网上很多方法，有人说时url的问题，有人直接建议把md和配置文件提出来，然后重新下载hexo。我试了好多种方法……

最后快崩溃了的时候有一个大哥提醒我了是不是更新的文章编写不规范？这我才去找我有没有更新文章……原来是我在更新主题的时候把一个之前我没更新的文章顺道更新了。导致了报错。

解决方法

我把不规范的那一篇博客先拿出来，再更新就好了。然后自己把博客编写规范再重挂上就好了。

最后 提醒一句：博客编写一定要规范阿！
