web简介

本文只记录部分web知识，只为记录一下。

web介绍

什么是web？

万维网
World Wide Web
由许多互相链接的超文本组成的
非常普遍的互联网应用
每天都有数以亿万计的Web资源传输：图片 html 音频 视频 快速传到浏览器上
我们平时通过浏览器上网都属于web，生活在web之中

web发展史

web1.0 初期
个人网站、门户网站 如www.163.com 提供信息给用户 但是不能添加修改

web2.0
微博、博客 如www.lofter.com www.weibo.com 可以人与人互动了

不同web时代 web安全问题也在变 web1.0时代主要针对web服务器，2.0时代逐渐针对web用户，且以XSS，CSRF为主

web流程

浏览器

浏览器得到域名后，需要向DNS服务器发出DNS请求去解析域名，DNS服务器接到域名后返回ip地址。这时浏览器可以找到web服务器。

通过http协议找web服务器，向服务器发送http请求，服务器处理完后会返回一个http
响应。

web通信

URL是什么？

  • 统一资源定位符
  • Uniform Resource Locator
  • 支持多种协议：HTTP、FTP……

    URL有什么用？

    定位服务器的资源

如何定位网站资源？
url就相当于网购的时候的收货地址，地址有一定的格式：省份/区市/道路/门牌号。URL同样有一定的格式。


什么是HTTP？

  • 超文本传输协议
  • Hyper Text Transfer Protocol
  • web通信时使用的协议
  • web的基础
  • 互联网应用最广的协议

    HTTP流程，请求，响应，报文

    HTTP到底报文长啥样？HTTP报文结构

    要注意头部和数据体直接有一个空行的。

HTTP的其他请求方式：

  • HEAD 与GET请求类似，不同在与服务器只返回HTTP头部信息，没有页面内容
  • PUT 上传指定URL的描述
  • DELETE 删除指定资源
  • OPTIONS 返回服务器支持的HTTP方法
  • Referer 也很重要，告诉服务器我们从哪里来 也就是 告知服务器该请求的来源 还可以用来判断来源合法性：防止盗链，防止CSRF漏洞

    web开发基础知识

    前端开发基础—HTML、JavaScript

    html太过基础，不细写了

    通常我们见到的js都是掺在html种的，那么哪些地方可以运行js？

在HTML的、html的事件属性种，如onclick，浏览器的js控制台

JavaScript BOM （操作浏览器）

  • 在控制台输入document.cookie可以获得cookie，document.cookie=“**”可以修改cookie
  • 在控制台输入(window.)screen，获取浏览器屏幕信息
  • 在控制台输入(window.)location，获取/控制用户页面URL
  • 在控制台输入(window.)navigator 获取访问者浏览器信息
  • 在控制台输入window open,close 可以操作浏览器窗口 例如：window.open(“http://www.baidu.com")

    web服务端环境

    web静态页面时期

    浏览器请求html，web服务端去相应的服务器找到html ，发送给浏览器

    web动态页面时期

    不同的是，动态页面由脚本驱动的，脚本文件（例如php文件）通过服务端的语言解释器的解析执行，最后解析成html文件，其他流程都跟静态一样了。在浏览器上时，不是html文件，而是后缀名为php的文件了。
