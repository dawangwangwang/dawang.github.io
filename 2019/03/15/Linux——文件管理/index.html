Linux目录结构

Windows：以多根的方式组织文件 C:\ D:\ E:\
Linux：以单根的方式组织文件 /

/sbin/service

存储普通用户家目录

文件管理之：创建/复制/移动/删除

创建文件 touch

touch file.txt //无则创建，有则修改时间
touch file3 file4
touch /home/file.txt
touch /home/{hhhhh,hehehe}
touch file{1..20}
touch wang{a,b,c}//{}集合，等价touch wanga wangb wangc

目录mkdir

mkdir dir1
mkdir /home/dir2 /home/dir3
mkdir /home/{dir3,dir4}
mkdir -v /home/{dir3,dir4}
mkdir -pv /home/dir3/dir4/111 //如果该路径中有的目录不存在 即父母目录，则一键创建了。不会有任何消息输出。

cp 复制

格式：
单源复制：
cp [OPTION]… [-T] SOURCE DEST（如果DEST不存在则创建，存在则覆盖）
多源复制：
cp [OPTION]… SOURCE… DIRECTORY（DEST必须为directory）

示例：
cp -if /data/[1-3].txt /data/test –test必须为目录，把三个文件一起复制到test中
cp -rf /data /practice –把data目录及目录下的内容一起复制到practice中

mv 移动

移动就是剪切。
mv file1 /home/dir3 将file1移动到/home/dir3
mv file2 /home/dir3/file20 将file2移动到dir3并改名字为file20
mv file4 file5 将file4重命名 当前位置的移动就是重命名

rm 删除

删除/home/dir
cd /home
rm -rf dir
-r 递归
-f force强制
-v 详细过程

当使用脚本删除的时候要使绝对路径：rm -rf /home/dir

rm -rf .pdf
rm -rf file
rm -rf * //不包含隐藏文件

文件时间

ls -l 1.php //仅看的是文件修改时间

Linux文件有四种时间

stat 1.php
//查看文件的详细属性（其中包括文件时间属性）

访问时间：atime，查看内容

修改时间：mtime，修改内容

改变时间：ctime，文件属性，比如权限

删除时间：dtime，文件被删除的时间

从企业版6开始relatime，atime延迟修改，必须满足其中一个条件：1.自上次atime修改后，已达到86400秒；2.发生写操作时；

注意

因为根目彔不开机有关，开机过程中仅有根目彔会被挂
载， 其他分割槽则是在开机完成乀后才会持续的迚行挂载的行为。就是因为如此，因此根目彔下不开
机过程有关的目彔， 就丌能够不根目彔放到丌同的分割槽去！那哪些目彔丌可不根目彔分开呢？有底
下这些：

  • /etc：配置文件
  • /bin：重要执行档
  • /dev：所需要的装置档案
  • /lib：执行档所需的函式库不核心所需的模块
  • /sbin：重要的系统执行文件

这五个目彔千万不能与根目彔分开在不同的分割槽！
