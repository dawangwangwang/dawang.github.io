Linux——文件管理
发表于 2019-03-14| 更新于 2019-03-17|Linux
阅读量:

文件管理
主要还是用shell命令对文件操作
创建、复制、删除、移动、查看、编辑、压缩、查找

Linux目录结构

Windows：以多根的方式组织文件 C:\ D:\ E:\
Linux：以单根的方式组织文件 /
[root@localhost ~]# ls /

根下的bin搬到usr下的bin里了

bin

/bin/ls,/bin/date

sbin

管理员使用的命令
/sbin/service

由此图可以看出来 ls是普通用户的命令，所以在bin里，而useradd是管理员才能操作的命令，所以该命令在sbin里

dev

设备文件 /dev/sda,/dev/sda1,/dev/tty1,/dev/tty2,/dev/pts/1,/dev/zero,/dev/null,/dev/random


root

root用户的HOME

home

存储普通用户家目录
ls /home/ 可以看到普通用户的家(Base)**

proc

虚拟的文件系统，反映出来的是内核，进程信息或实时状态 **

查看除sshd以外的进程：

usr

系统文件，相当于C:\Windows

/usr/local 软件安装的目录，相当于C:\Program

/usr/bin 普通用户使用的应用程序

/usr/sbin 管理员使用的应用程序

/usr/lib 库文件Glibc 32bit

/usr/lib64 库文件Glibc 64bit

boot

存放的系统启动相关的文件，例如Kernel，grub（引导装载程序）

grub 开机的时候负责把系统启动的

etc(重头戏)配置文件

系统相关如网络 ：
/etc/sysconfig/network
/etc/hostname 主机名
应用相关配置文件如：
/etc/ssh/sshd_config…

tmp 临时文件

全局可写，进程运行产生的临时文件

var

存放的是一些变化文件，比如数据库，日志，邮件……
mysql： /var/lib/mysql
vsftpd: /var/ftp
mail: /var/spool/mail
cron: /var/spool/cron
log: /var/log
临时文件: /var/tmp(进程产生的临时文件)

文件路径

路径的作用：定位文件

有一个命令：tree
tree -L 2 /etc 可以查看/etc下的文件的两级

/home/abc/file1 和/home/def/file1 不一样！
/abc/file1 和abc/file1 可能不一样

绝对路径:
从/开始的路径，文件完整的路径 /home/abc/file1
~表示/home 我家

相对路径:
相对于当前目录开始，不是完整的路径，从此时所在目录为起点 a.txt ./a.txt ../bob/b.txt

cd 改变目录
cd - 返回
cd .
cd ..

文件管理之：创建/复制/移动/删除

创建文件 touch

touch file.txt //无则创建，有则修改时间
touch file3 file4
touch /home/file.txt
touch /home/{hhhhh,hehehe}
touch file{1..20}
touch wang{a,b,c}//{}集合，等价touch wanga wangb wangc

目录mkdir

mkdir dir1
mkdir /home/dir2 /home/dir3
mkdir /home/{dir3,dir4}
mkdir -v /home/{dir3,dir4}
mkdir -pv /home/dir3/dir4/111 //如果该路径中有的目录不存在 即父母目录，则一键创建了。不会有任何消息输出。

cp 复制

格式：
单源复制：
cp [OPTION]… [-T] SOURCE DEST（如果DEST不存在则创建，存在则覆盖）
多源复制：
cp [OPTION]… SOURCE… DIRECTORY（DEST必须为directory）

Code

示例：
cp -if /data/[1-3].txt /data/test –test必须为目录，把三个文件一起复制到test中
cp -rf /data /practice –把data目录及目录下的内容一起复制到practice中

mv 移动

移动就是剪切。
mv file1 /home/dir3 将file1移动到/home/dir3
mv file2 /home/dir3/file20 将file2移动到dir3并改名字为file20
mv file4 file5 将file4重命名 当前位置的移动就是重命名

rm 删除

删除/home/dir
cd /home
rm -rf dir
-r 递归
-f force强制
-v 详细过程

当使用脚本删除的时候要使绝对路径：rm -rf /home/dir

rm -rf .pdf
rm -rf file
rm -rf * //不包含隐藏文件

文件管理之——查看文件内容

文本文件（cat less head tail tailf grep……）

cat /etc/hosts

-n 显示行号
-A 包括控制字符（换行符/制表符）
Linux $
windows ^M$
例：当b.txt被windows记事本修改后，在Linux打开本应$会是^M$，所以需要下载东西，转换文档格式

yum -y install dos2unix  
dos2unix b.txt  
cat -A b.txt

head

head /etc/passwd 默认看前10行
head -2 /etc/passwd

tail

tail /etc/passwd 尾部后十行
tail -1 /etc/passwd
tail /var/log/messages 查看日志后十行，也就是最新日志
tail -f /var/log/messages 一直动态监控文件尾部变化

less

可以分页显示
less /var/

grep

grep ‘root’ /etc/passwd 搜索文件中带root的行
grep ‘^root’ /etc/passwd
grep ‘root$’ /etc/passwd

vi编辑器（可视化编辑器）

文件编辑器 gedit（图形化编辑器）
文件编辑器 vi，vim，nano
[root@localhost ~]# vim 1.txt
刚打开是，命令模式
按i可以进入编辑模式
按v、V、^v进入可视化编辑模式
按:进入扩展命令模式
查看完以后按Esc+:w 保存
查看完以后按Esc+:wq 即可保存并退出

命令模式

  1. 光标定位
    h(左) j(下) k(上) l(右)
    0（移到行首） $（移到行尾）
    gg (移到第一行的第一个) G(移动到最后一行)
    3G(移动到第三行)
    /string (n往后找，N往前找)
    /^字母 找以哪个字母为开头的
    /字母$ 找以哪个字母为结尾的
  2. 文本编辑（少量）

y 复制
yy（复制一行）3yy（复制三行） ygg（从光标位置复制到第一行） yG(从光标位置复制到最后一行)

d 删除
dd 3dd dgg dG（以行为单位）

p 粘贴

x 删除光标所在的字符

D 从光标处删除到行尾

u undo撤销

^r redo重做（反撤销）

r 可以用来修改一个字符（不用进入编辑模式了）

3.进入其他模式
a 进入插入模式

i
o 另起一行进入插入模式
A 进入到行末进入插入模式

: 进入末行模式（扩展命令模式）
v 进入看视模式
^v 进入可视块模式
V 进入可视行模式
R 进入替换模式

插入模式

^p

可视块模式

块插入（在指定块前加入字符串）：选择块，I 在块前插入字符，ESC

块替换： 选择块，r 输入替换的字符

块删除： 选择块，d|x

块复制： 选择块，y

扩展命令模式

a.保存退出

:10 进入第十行
:w 保存
:q 退出
:wq 保存并退出
:w! 强制保存
:q! 强制退出，不保存
:wq! 强制保存并退出
:x 保存并退出
:X 设置密码 （若第二个人不知道密码，还可以给文件加第二次密，导致第一个人也打不开） shift+zz也可以在命令模式保存

b.查找替换
:范围 s/old/new/选项

:1,5 s/root/wang
从1-5行的root替换成wang

:5,$ s/root/wang/g
$表示最后一行

:1,$ s/root/wang/g
=: s/root/yang/g %表示全文 g表示全局

:% s#/dev/abc#/var/ccc#g
这次以#代替/，而/是为了表达路径

:,6 s/root/wang/
当前行到第6行，root替换为wang

:4,9 s/^#//
4-9行的开头#替换为空

:2,10 s/.*/#&/
2-10前加入#字符（.*整行，&引用查找的内容）

c.读入文件/写文件（另存为）
:w
存储到当前文件夹

:w /tmp/a.txt
另存为/tmp/a.txt

:1,3 w /tmp/b.txt
1-3行存到/tmp/b.txt

:r /etc/hosts
读入文件到当前行后

:5 r /etc/hosts
读入文件到第5行后

:set nu

:set ai
自动缩进

:set list
显示控制字符

:set nonu
取消设置行号

:set noic

/etc/vimrc
影响所有系统用户

~/.vimrc
影响某一个用户

打开文件：
vim ~/.vimrc
set ic

补充：

vim -O 1.php 1.txt
同时打开两个文件，横向展示，^ww切换

文件时间

ls -l 1.php //仅看的是文件修改时间

Linux文件有四种时间

stat 1.php
//查看文件的详细属性（其中包括文件时间属性）

访问时间：atime，查看内容

修改时间：mtime，修改内容

改变时间：ctime，文件属性，比如权限

删除时间：dtime，文件被删除的时间

从企业版6开始relatime，atime延迟修改，必须满足其中一个条件：1.自上次atime修改后，已达到86400秒；2.发生写操作时；

文件类型

通过颜色判断文件类型不一定对
Linux扩展名改变不了文件本质

方法一：

ls -l 文件名 //看第一个字符
- 普通文件（文本，二进制，压缩文件，电影，图片……）

d 目录文件（蓝色）

b 设备文件（块设备）存储设备硬盘，U盘 /dev/sda,/dev/sda1

c 设备文件（字符设备）打印机，终端 /dev/tty1

s 套接字文件

p 管道文件

l 链接文件（淡蓝色）

du -sh /etc //查看etc大小

方法二：

file /etc/hosts 查看host文件类型

最后

type 查看命令类型类型，是alias呢还是内置命令呢，还是某个文件
type ls
type 1.php
type -a ls//-a列出该命令所有类型

file查看文件类型，例如文本文件还是二进制文件，管道文件，设备文件，链接文件
file /dev/sda

stat 文件的属性，例如文件的名称，大小，权限，atime，ctime，mtime
stat /etc/hostname

注意

因为根目彔不开机有关，开机过程中仅有根目彔会被挂
载， 其他分割槽则是在开机完成乀后才会持续的迚行挂载的行为。就是因为如此，因此根目彔下不开
机过程有关的目彔， 就丌能够不根目彔放到丌同的分割槽去！那哪些目彔丌可不根目彔分开呢？有底
下这些：

  • /etc：配置文件
  • /bin：重要执行档
  • /dev：所需要的装置档案
  • /lib：执行档所需的函式库不核心所需的模块
  • /sbin：重要的系统执行文件

这五个目彔千万不能与根目彔分开在不同的分割槽！
