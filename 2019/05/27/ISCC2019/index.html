这题跟西电网信杯线下赛的一道题类似
需要绕过的点是：
a.$value[$i] > 32 && $value[$i] < 127

value数组里的数要大于32，小于127，但事实要想得到理想字符，数组里的数都是介于32到127之间的。

b.$username == 'w3lc0me_To_ISCC2019' && intval($password) < 2333 && intval($password + 1) > 2333

要让username==字符串，全靠value。那么后边intval怎么绕过？

如何绕过：
a:

  • chr函数在转换时会自动取模256,所以我们只需要在原本ascii码基础上+256即可绕过

b:

  • intval()在处理16进制时存在问题,但强制转换时是正常的，也就是说我们普通数字绕不过，我们可以用十六进制代替反正intval可以强制转换得到想要的数字.

这题也是代码审计

通过审计代码我们了解到我们应该get传过去两个变量：
action、key
action===auth,而key经过加密后要与$hashed_key相等。要满足等式$hashed_input!==$hashed_key,flag才会出来。$hashed_input由key决定可以改变，而$hashed_key是固定的。
所以变量覆盖$hashed_key=我的key加密出的数据
payload：
?action=auth&hashed_key=2ac81299578013795576682809a4a16d1ecca67118a797ea576c113c41786849&key=wang

flag{7he_rea1_f1@g_15_4ere}
